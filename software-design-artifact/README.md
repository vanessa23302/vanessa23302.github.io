# Software Design and Engineering Artifact

## Artifact Overview
This artifact is the Contact Service application that was originally developed in CS 320 Software Testing, Automation, and Quality Assurance. The application is written in Java and demonstrates object-oriented design through the use of multiple classes that manage and validate contact information.

The project includes the following components:

- Contact.java: Defines the contact object and enforces validation rules for fields such as ID, name, phone number, and address.
- ContactService.java: Manages the creation, storage, updating, and deletion of contact objects.
- ContactTest.java: Contains JUnit tests used to verify the correct behavior of the Contact and ContactService classes.

## Enhancement
The artifact was enhanced to strengthen the validation logic and improve the structure of the service class responsible for managing contact records. The enhancement ensures that contact fields follow specific constraints such as maximum length and non-null requirements.

Additional unit tests were included to verify that invalid data is rejected and that valid data is stored correctly.

## Skills Demonstrated
This artifact demonstrates several important software engineering concepts including:

- Object-oriented programming in Java
- Class design and encapsulation
- Data validation and error prevention
- Unit testing using JUnit
- Separation of responsibilities between classes

These improvements help ensure that the application behaves predictably and maintains valid data, which is a key requirement in professional software development.
